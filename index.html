⚠️ Security Exploit Tests

Each section below maps to a finding in the Security Audit Report. All tests must be run from inside the SDK WebView with this domain whitelisted. Open Safari → Develop → [Device] → [WebView] to observe console output.

C-01: Unencrypted Cookie Storage

Creates cookies that the SDK will persist as a plaintext .plist file on disk.
After clicking "Create", do: Xcode → Window → Devices → Download Container → AppData/Documents/PhonePeWebView/Cookies/ → open any .plist in TextEdit.

C-02: JS Injection — Backslash & Newline Escape Bypass

Sends a malicious callbackId to the location bridge.
The SDK escapes ' → \' but NOT \, so a leading backslash cancels the escape and breaks out of the JS string.
Confirmation: An alert() popup fires inside the WebView.

Payload A: \'; alert("C02_INJECTED: "+document.cookie); //

Payload B: raw newline in callbackId breaks the string literal

C-03: Custom Bridge Executes Raw JavaScript

Sends a crafted callbackId to the CUSTOMTEST bridge.
The bridge builds callbackId(response) and executes it verbatim via evaluateJavaScript.
Confirmation (Test A): document.title changes to C03_PWNED.
Confirmation (Test B): alert() shows real cookie values.

—

C-04: AUTH Bridge Auto-Granted to Every Whitelisted Domain

Calls fetchAuthToken and fetchGrantToken directly.
Neither is listed in this domain's entityPermissions, but AUTH is in the SDK's hardcoded universalBridge set — so the permission check passes.
Confirmation: Any callback fires (even a stub/empty response proves the permission passed).

H-02: Unrestricted iframe — Non-Whitelisted Domain Loads

Injects a hidden <iframe> pointing to httpbin.org (not in any whitelist).
The SDK's navigation handler returns .allow for all sub-frames before any domain validation.
Confirmation: The iframe's onload fires — visible in the status box and Network tab of Web Inspector.

H-03: Full GPS Coordinates Exposed to JS

Intercepts the getCurrentLocation callback and logs the raw response.
The SDK passes CLLocation.dictionaryRepresentation() directly — full precision, no rounding.
Confirmation: Location data in the status box shows 6+ decimal places of precision.

H-04: Camera Permission Triggers Photos Dialog (Not Camera)

Calls seekPermission(["CAMERA"]) via the SDK bridge.
The SDK calls PHPhotoLibrary.authorizationStatus() instead of AVCaptureDevice.
Confirmation: iOS shows "Would Like to Access Your Photos" instead of a camera dialog.
(Uses the existing SDK method from bundle.js)

Watch the iOS system dialog — it should say "Photos" not "Camera".

H-06: Full URL + Query Params Sent to Analytics

Navigates to the whitelisted domain with a fake token and session in the URL.
logPageLoadEvent() passes the full URL (no query stripping) to analyticsProvider.sendEvent.
How to confirm: Set a breakpoint at analyticsProvider?.sendEvent(eventType: .pageLoad(url: url)) in PhonePeWebViewNavigationDelegate.swift line 259, then click the button.
In the debugger run: po url?.absoluteString — the token will be visible.

Set Xcode breakpoint first, then click.

H-07: WebRTC Capture Type Ignored — Audio-Only Grants Camera Too

Step 1: Request audio only. Step 2: Request video only without asking.
The SDK's decideMediaCapturePermissionsFor ignores the WKMediaCaptureType parameter and returns .grant for everything.
Confirmation: Both steps get a live stream with the requested tracks granted.

H-08: Unknown Permission Names Auto-Approved as true

Sends completely fabricated permission strings directly via the message handler.
The SDK's guard falls through and appends permissionGranted: true for every unknown name.
Confirmation: Status box shows all fabricated permissions returned as permissionGranted: true.

javaScriptCanOpenWindowsAutomatically: true is set in your CRISP config.
A 2-second delayed window.open() fires with no user interaction.
Confirmation: A new WebView window opens automatically. Up to maxWebViewSupported: 2 windows allowed.

M-05: TrapURL "www.google.com" is Unescaped Regex — False Positive

Your CRISP config has trapUrls[1] = "www.google.com" — a plain string used as a regex.
The dots . match any character, so www-google-com.evil.io also triggers the trap.
Confirmation (Test A): The non-google URL triggers onTrapUrlHit.
Confirmation (Test B): 10 rapid iframe injections — watch Xcode CPU Navigator for repeated regex compilation spikes.
